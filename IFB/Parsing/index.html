    Parsing Fields

    Overview

    When working with fields in code inside the Input Form Builder there may be times that we need to use the value of a field for a calculation or for comparison. If we use Macro.GetField("ID") then a value of a formatted string is returned no matter the field's data type. If we use Loan.Fields("ID").Value then Input Form Builder uses the field's data type for the object returned but this may be inconsistent with how the field is used, an example of this is when a string field is used to store the datetime a button was clicked vs a date field. To help with this the Field class has methods that try and parse field values into a data type that can be used. Below are examples on how to use these parsing methods.

    Note

    When using Loan.Fields("ID").Value if the field is empty then for string based field type (String, YN, Checkbox, Zipcode, State, Phone, SSN, Dropdown, DropdownList) the returned value will be a empty string object. For Decimal, Integer, Date, and MonthDay the return type will be a Nothing or Null object.

    DateTime

    The ToDate() method can be used to try and parse a field into a DateTime object. If the value cannot be parsed then a value of DateTime.MinValue (01/01/0001 12:00:00 AM) is returned. Below show a example of using the ToDate() method; first using it to check to see if the value is empty (or returned the MinValue), followed by checking to see if the day is greater than or equal to a specific date.

If myDate = DateTime.MinValue Then
    'do something based on empty date

If myDate >= #3/1/2024# Then

    Integer

    Using the ToInt() method will try and parse a field into a Integer object. If the value cannot be parsed then a value of 0 will be returned. Below shows an example of using the ToInt() method.

    Note

    When checking for an empty field you may not want to parse the field to a integer as a null or nothing integer will be 0. This means we cannot use is equal to 0 to test if the field has been filled out with a 0 as the value or is truly a blank field. In this case it may be better to use the IsEmpty() method first.

    'do something based on a blank or empty field
End If

'to zero. Note that if the field is blank
'then this will return true as when a empty or
'blank field is parsed the value returned is 0
If myInt.ToInt() >= 0 Then
    'do something based on parsed value
End If

    String

    The ToString() method will return the string value of the field. In the case the field is blank or empty an empty string object will be returned. A field object also has a Formatted and Unformatted value property, which will return a string value based on what property is called. A Formatted value returned is a value that is easier for users to read, while a Unformatted value is a value with no formatting. The example below shows what values are returned based on the fields data type when the Formatted or Unformatted property is used.
