    Validate Data Entry

    Overview

    Sometimes we need to verify that data has been entered into a control or field before something else can happen on a form. We can do this in various ways depending on how many and what type of data we are verifying.

    Single Control

    If we need to verify that a control has a value before something happens we can use the below to check if the value of the control is not empty.

    Note

    Below I am just checking if a Textbox control has a value, and while most controls have a value property there are some that may not. Make sure to check and adjust as needed to your particular control type.

    1
2
3
4
    'txtTextTest is a textbox control
If txtTextTest.Value  = Nothing Then
    Msgbox("No value found", 48, "No value")
End If

if (txtTextTest.Value == "")
{
    Macro.Alert("No value found.");
}

    Multiple Controls

    Same Control Type

    If we need to verify multiple controls and they happen to be of the same control type we can use something like the below to verify if the set of controls have data.

    Note

    The below assumes that each control is mapped to the same field type and the field type is blank when no value is set. For controls that are mapped to field types that default a value, like the date field type defaulting to // when empty, check further down.

'value is set to nothing
For Each tb as Textbox in  textboxes
    'Value is nothing, alert the user or

    Various Field Types

    If we have various control types and various field types we can use the below to check if they are blank. What the below does is takes a list of controls and then gets the field bound to the control. Based on that fields type we can implement a check to see if the field is considered blank.

    Note

    The below example oh take in account for String, Decimal_2, and Date field types. More field types can be found in the EncompassObjects Help file.

    'Declare a array of controlIDs that we want to verify
Dim controls() as String  = {"txtTextTest", "txtDcmlTest", "txtDateTest"}

'Create a boolean to hold if data is missing
Dim missing_data as Boolean = False

'Loop through controls and check mapped fields for
'missing data
For Each control as String in controls
    'Find the control based on the Control ID
    Dim ctrl = FindControl(control)
    'Get the field mapped to the control
    Dim field as FieldDescriptor = ctrl.Field
    'Get the value of the field
    Dim field_value = Macro.GetField(field.FieldID)

    'Based on the fields type check if empty
    Select Case field.Format
        'Format of field is String
        Case 101
            If String.IsNullorEmpty(field_value) Then
                missing_data = True
            End If
        'Format of field is Decimal_2
        Case 203
            'Can add custom checks like treating 0.00 as an empty
            'and not just checking for a blank field
            If String.IsNullorEmpty(field_value) or field_value = "0.00" Then
                missing_data = True
            End If
        'Format of field is Date
        Case 301
            If field_value = "//" Then
                missing_data = True
            End If
    End Select
Next

'If any data is missing notify the user or
'do some other action
If missing_data = True Then
    Macro.Alert("Data is missing please verify.")
End If

    //Declare a array of controlIDs that we want to verify
string[] controls = {"txtTextTest", "txtDcmlTest", "txtDateTest"};

//Create a boolean to hold if data is missing
bool missing_data = false;

//Loop through controls and check mapped field for
//missing data
foreach (string control in controls)
{
    //Find the control based on Control ID
    var ctrl = (FieldControl)FindControl(control);
    //Get the field mapped to the control
    FieldDescriptor field = ctrl.Field;
    //Get teh value of the field
    string field_value = Macro.GetField(field.FieldID);

    //Based on the fields type check if empty
    switch (field.Format)
    {
        //Format of field is String
        case LoanFieldFormat.STRING:
            if (String.IsNullOrEmpty(field_value))
            {
                missing_data = true;
            }
            break;
        //Format of field is Decimal_2
        case LoanFieldFormat.DECIMAL_2:
            //Can add custom check like treaking 0.00 as an empty
            //and not just checking for a blank field
            if (String.IsNullOrEmpty(field_value) || field_value == "0.00")
            {
                missing_data = true;
            }
            break;
        //Format of field is Date
        case LoanFieldFormat.DATE:
            if (field_value == "//")
            {
                missing_data = true;
            }
            break;
    }
}

//If any data is missing notify the user or
//do some other action
if (missing_data == true)
{
    Macro.Alert("Data is missing please verify.");
}
